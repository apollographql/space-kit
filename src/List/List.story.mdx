# List

**Lists** provide a way for sures to interact with the application by selecting from a set of options. While thatâ€™s simple in theory, lists can be complex and not eveything is detailed below. lists can emerge from Select buttons or a simple button.

## Simple List

The simple list provides a list of options for a user to choose from and updates the button or select button with the choice the user has selected.

## Segmented List

The segmented list provides a way to group related options. This can be achieved
by a divider and in other cases, headers can be used to group similar
information and provide a nice visual grouping.

## `padding`

Padding will determine how spaced the insides of the element is.

Padding is used to calculate the margin between elements and to add a negative margin to the top and bottom of `List`. This presents a gotcha: we don't know if you're going to have collapsable margins so we don't know if we should add the real amount or half of the desired margin. We're assuming margin collapsing is not available.

## `as`

Specify the element used to render the button with the `as` prop. The default is `<div />`.

All props added to the `as` element will be merged with the props added by the component. `classNames`, `styles`, and `css` will be intelligently merged.
